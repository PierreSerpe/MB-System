26  * to store multibeam data read from the MBF_HSLDEDMB format (MBIO id 22).
27  *

29  * Date: January 20, 1993

31  */
32 /*
33  * Notes on the MBF_HSLDEDMB data format:
34  * 1. This data format is used to store 59 beam Hydrosweep DS bathymetry
35  * data. This format was created by Dale Chayes of L-DEO and Dan
36  * Chayes of NRL for use with an early version of a ping editor.
37  * Most data files in this format consist of Hydrosweep DS data
38  * collected on the R/V Maurice Ewing or the R/V Thomas Thompson.
39  * 2. The data consist of 328 byte records including 1-byte characters,
40  * 2-byte integers, and 8-byte integers.
41  * 3. The 59 depth values are stored centered in 59 value arrays. The
42  * center beam is in word 30 of the depth and distance arrays.
43  * 4. There is no provision for embedding comments in the data.
44  * 4. Comments can be embedded in the data as 328 byte ascii strings,
45  * where the first four characters must always be "zzzz" so that the
46  * seconds value is 2054847098.
47  * 5. We expect the use of this format to be superceded by format
48  * MBF_MBLDEOIH.
49  *
50  * The kind value in the mbf_hsldedmb_struct indicates whether the
51  * mbf_hsldedmb_data_struct structure holds data (kind = 1) or an
52  * ascii comment record (kind = 2).
53  *
54  * The mbf_hsldedmb_data_struct structure is a direct representation
55  * of the binary data structure used in the MBF_HSLDEDMB format.
56  */
