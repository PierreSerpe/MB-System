25 /*
26  * mbf_hsmdaraw.h defines the data structures used by MBIO functions
27  * to store multibeam data read from the MBF_HSMDARAW format (MBIO id 101).
28  *
29  * Author: Dale Chayes
30  * David W. Caress
31  * Date: August 11, 1995
32  *
33  *
34  *
35  * Notes on the MBSYS_HSMD data structure:
36  * 1. The Atlas Hydrosweep MD generates several types of data files
37  * including:
38  * "*.P" Profile data files that contain bathy data
39  * "*.W" Wreck data files with info about obstructions
40  * "*.R" Raw data files. At present, this is the only
41  * data file format that is supported by MB. These
42  * files are written in XDR format. Code to assist
43  * in parsing these files was provided by STN Atlas.
44  *
45  * 2. The MD systems output 40 beams of bathymetry on one side and
46  * 160 beams of amplitude measurements, along with a moras of other
47  * information. MD systems ping to one side (making 40 beams & 160
48  * sidescan measurements), wait for all the returns and then ping
49  * to the other side. The "Raw.beamside" variable indicates which side
50  * the current ping is pointed.
51  *
52  * 3. The records all include navigation and time stamp information.
53  * There is a Header block in front of every data record. The
54  * record types are:
55  * - Raw Event (RAW in the case statement) which contains the
56  * travel time and amplitude data along with other interesting
57  * parameters.
58  * - Navigation (NAV -------"------------) which contains one
59  * input record from the NAV system.
60  * - MD Event
61  * - Beam angle data (ANG) which contains the pointing angles
62  * of the beams. The MD system is supposed to be able to operate
63  * in a beam hopping mode where the beam angles shift from ping
64  * to ping, but this is not yet enabled. The Raw.noho variable
65  * indicated the "hopping" state.
66  * - Sound Velocity (SVP) can contain a sound speed profile.
67  * - Raw Event (REV) which seems to happen at the beginging of
68  * each data file.
69  *
70  * 4. A single ping usually insonifies one side of the track and results
71  * in RAW data record. Sequential RAW records occur for alternating
72  * sides. NAV records appear to happen at the rate at which they are
73  * supplied by either of the possible Navigation inputs. In early legs
74  * (1994) on the Alliance, two different nav systems were inserting
75  * records, one flagged with an "I" for Integrated Nav and the other
76  * using a "G" to signify GPS. Some time early in 1995, the onboard
77  * nav system was changed and now there appears to be only one type
78  * "I" of input nav data. It appears to happen once per second.
79  *
80  * 5. The data structure defined below includes all of the values
81  * which are passed in Hydrosweep MD records.
82  *
83  * 6. Comment records have been implemented for use with MB-System
84  * programs. Comment records are NOT part of the vendor format
85  * definition and are NOT supported by Atlas software. Using
86  * mbcopy with the -N option will remove all comments and make
87  * the data compatible with Atlas software.
88  */
