44 /*
45  * Notes on the MBSYS_ATLAS data structure:
46  *
47  * 1) STN Atlas Marine Electronics (aka SAM) sonars write raw data in real-time
48  * as binary XDR encoded data. Files are stored on disk by the HYDROMAP
49  * Online workstation. The workstation on the Ewing is an HP Vectra
50  * running SuSe Linux (2.2 kernel.)
51  *
52  * 2) Files are 10 minutes long.
53  *
54  * 3) Multiple (parallel files are created). For example:
55  *
56  * 64k 00010529004010.adt
57  * 28k 00010529004010.ang Angle data
58  * 28k 00010529004010.cdt Sound speed profile(s)
59  * 8.0k 00010529004010.evt Event data, comments, etc.
60  * 344k 00010529004010.fsw Sounding (travel time, sidescan, backscatter...
61  * 148k 00010529004010.lev Level (tide gage) data
62  * 664k 00010529004010.lot center beam (telegram 26) and lots of zeros
63  * 60k 00010529004010.mot Motion sensor (attitude)
64  * 388k 00010529004010.nav Navigation sensors
65  *
66  * lot file: 26 (center beam) and lots of zeros
67  *
68  * For starters (June, 2001) we are assuming fixed beam spacing w/ a 90 or
69  * 120 degree swath so all of the swath mapping data we need is in the ".fsw"
70  * data file.
71  *
72  * 4) Data record type ids:
73  * MB-System critical records are:
74  *
75  * Sounding data 10 T0, nav, attitude,
76  * Depths 11 travel time+amp/beam
77  * Sidescan 12 Multiple packets of sidescan data
78  * Tracking windows 19
79  * Amplitude 28 TX and RX data necessary for calculating true amp
80  *
81  * Other potentially interesting things:
82  * "Center beam" 26
83  *
84  * 5) Time values are in Unix seconds (seconds since 1/1/1970 00:00:00
85  */
