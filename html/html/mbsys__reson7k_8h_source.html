34 /*
35  * Notes on the mbsys_reson7k data structure:
36  * 1. This format is defined by the Interface Control Document
37  * for RESON SeaBat 7k format v0.42.
38  * 2. Reson 7k series multibeam sonars output bathymetry, per beam
39  * amplitude, and sidescan data.
40  * 3. The Reson 6046 datalogger can also log sidescan and subbottom
41  * data from other sonars.
42  * 4. The 7k record consists of a data record frame (header and checksum),
43  * a record type header, an optional record data field and an optional
44  * data field for extra information. The optional data field typically
45  * holds non-generic sensor specific data. *
46  * 5. Navigation data may be found in three different record types.
47  * The bathymetry records (kind = MB_DATA_DATA) hold navigation
48  * and attitude data, but these values are not initially set by
49  * the Reson 6046 datalogger. In MB-System these values get set
50  * by running the program mb7kpreprocess by interpolating the
51  * the values found in either the R7KRECID_Position records
52  * (kind = MB_DATA_NAV1) or the R7KRECID_Bluefin records
53  * (kind = MB_DATA_NAV2). MB-System uses the bathymetry records as
54  * the primary navigation source, so the interpolated values are
55  * accessed by mbnavedit and, by default, mbnavlist. The raw values
56  * of the ancillary navigation records (R7KRECID_Position and
57  * R7KRECID_Bluefin) may be accessed by mbnavlist using the -N1
58  * and -N2 options, respectably.
59  * 6. Attitude data may be found in three different record types.
60  * The bathymetry records (kind = MB_DATA_DATA) hold navigation
61  * and attitude data, but these values are not initially set by
62  * the Reson 6046 datalogger. In MB-System these values get set
63  * by running the program mb7kpreprocess by interpolating the
64  * the values found in either the R7KRECID_RollPitchHeave records
65  * (kind = MB_DATA_ATTITUDE) or the R7KRECID_Bluefin records
66  * (kind = MB_DATA_NAV2). MB-System uses the bathymetry records as
67  * the primary attitude source, so the interpolated values are
68  * accessed by mbnavedit and, by default, mbnavlist. The raw values
69  * of the secondary ancillary navigation records (R7KRECID_Bluefin),
70  * including attitude, may be accessed by mbnavlist using the -N2
71  * option.
72  * 7. The MB-System code assumes that a Reson 7k data file will include
73  * either R7KRECID_RollPitchHeave and R7KRECID_Position records
74  * or R7KRECID_Bluefin records. Bad things will happen if the
75  * data file contains both the generic records and the bluefin
76  * records.
77  */
