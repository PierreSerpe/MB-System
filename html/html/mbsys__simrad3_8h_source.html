25  * mbsys_simrad3.h defines the MBIO data structures for handling data from
26  * new (post-2006) Kongsberg multibeam sonars (e.g. EM710, EM3002, EM302, EM122).
27  * The data formats associated with Kongsberg multibeams
28  * (both old and new) include:
29  * MBSYS_SIMRAD formats (code in mbsys_simrad.c and mbsys_simrad.h):
30  * MBF_EMOLDRAW : MBIO ID 51 - Vendor EM1000, EM12S, EM12D, EM121
31  * : MBIO ID 52 - aliased to 51
32  * MBF_EM12IFRM : MBIO ID 53 - IFREMER EM12S and EM12D
33  * MBF_EM12DARW : MBIO ID 54 - NERC EM12S
34  * : MBIO ID 55 - aliased to 51
35  * MBSYS_SIMRAD3 formats (code in mbsys_simrad2.c and mbsys_simrad2.h):
36  * MBF_EM300RAW : MBIO ID 56 - Vendor EM3000, EM300, EM120
37  * MBF_EM300MBA : MBIO ID 57 - MBARI EM3000, EM300, EM120 for processing
38  * MBSYS_SIMRAD3 formats (code in mbsys_simrad3.c and mbsys_simrad3.h):
39  * MBF_EM710RAW : MBIO ID 58 - Vendor EM710
40  * MBF_EM710MBA : MBIO ID 59 - MBARI EM710 for processing

48 /*
49  * Notes on the MBSYS_SIMRAD3 data structure:
50  * 0. Kongsberg was formerly known as Simrad; the names Kongsberg
51  * and Simrad are interchangeable within MB-System source code
52  * and documentation.
53  * 1. Kongsberg multibeam systems output datagrams which are
54  * a combination of ascii and binary. This code has been written
55  * using data format specifications found in Kongsberg document
56  * 16092ai EM Series Data Formats (rev I January 20, 2006
57  * through Rev Q February 2013).
58  * 2. Kongsberg multibeam sonars output both bathymetry
59  * and amplitude information for beams and sidescan information
60  * with a higher resolution than the bathymetry and amplitude.
61  * 3. This code and formats 58 and 59 support data from the
62  * EM-710 multibeam sonar and later models. Data from older
63  * Kongsberg multibeams are supported by formats 51, 56, and 57.
64  * 4. Each telegram is preceded by a two byte start code 0x02 and
65  * followed by a three byte end code consisting of 0x03
66  * followed by two bytes representing the checksum for
67  * the data bytes. MB-System does not check the checksums
68  * on input, but does calculate the checksums for output.
69  * 5. The Kongsberg datagram format manual lists a large number
70  * of datagram types. The complete list of telegram start codes,
71  * types, and sizes is given below. Datagram listings preceded
72  * by an "*" are recognized by MB-System. Unrecognized datagrams
73  * will be skipped on input and not included in output files.
74  *
75  * Multibeam Datagrams:
76  * *0x0244: Bathymetry 48-4092 bytes
77  * *0x0258: Bathymetry (EM710 and later) variable size
78  * 0x024B: Central beams echogram variable size
79  * *0x0265: Raw range and beam angle (deprecated) 112-1658 bytes
80  * *0x0246: Raw range and beam angle "F" 24-2056 bytes
81  * *0x0266: Raw range and beam angle "f" 44-1658 bytes
82  * *0x024E: Raw range and beam angle "N" (EM710 and later) variable size
83  * *0x0253: Sidescan 48->5K bytes
84  * *0x0259: Sidescan (EM710 and later) variable size
85  * *0x026B: Water column variable size
86  * *0x024F: Quality factor variable size
87  * *0x02E1: Bathymetry (MBARI format 57) 48-4092 bytes
88  * *0x02E2: Sidescan (MBARI format 57) 48->5K bytes
89  * *0x02E3: Bathymetry (MBARI format 59) 48-4092 bytes
90  * *0x02E4: Sidescan (MBARI format 59) 48->5K bytes
91  * *0x02E5: Bathymetry (MBARI format 59) 48-4092 bytes
92  *
93  * External Sensor Datagrams
94  * *0x0241: Attitude Output 1222 bytes
95  * *0x026E: Network attitude variable size
96  * *0x0243: Clock Output 28 bytes
97  * *0x0268: Height Output 24 bytes
98  * *0x0248: Heading Output 422 bytes
99  * *0x0250: Position 100-134 bytes
100  * 0x0245: Single beam echosounder depth 32 bytes
101  * *0x0254: Tide Output 30 bytes
102  * 0x0273: Surface sound speed (deprecated) variable size
103  * *0x0247: Surface sound speed variable size
104  * *0x0256: Sound velocity profile (deprecated) variable size
105  * *0x0255: Sound velocity profile variable size
106  * 0x0257: SSP input variable size
107  *
108  * Multibeam Parameters
109  * *0x0249: Parameter - Start variable size
110  * *0x0269: Parameter - Stop variable size
111  * 0x0270: Parameter - Remote variable size
112  * *0x0252: Runtime Parameter 52 bytes
113  * *0x024A: Mechanical transducer tilt variable size
114  * *0x0233: Extra Parameter variable size
115  * *0x0230: Processing Unit ID 108 bytes
116  * *0x0231: Processing Unit Status 85 bytes
117  * *0x0232: Processing Unit BIST variable size
118  *
119  * 6. Kongsberg systems record navigation fixes using the position
120  * datagram; no navigation is included in the per ping data. Thus,
121  * it is necessary to extrapolate the navigation for each ping
122  * at read time from the last navigation fix. The frequency of
123  * GPS fixes generally assures that this is not a problem, but
124  * we offer no guarantees that this will always be the case.
125  * In this format the navigation fix datagrams include copies of
126  * the ASCII data records (typically NMEA 0183) input into the
127  * sonar by the navigation system.
128  * 7. The beam depths are give relative to the transmit transducer
129  * or sonar head depth and the horizontal location of the active
130  * positioning system's antenna (or reference point). Heave,
131  * roll, pitch, sound speed at the transducer depth and ray
132  * bending have been applied.
133  * 8. The new Kongsberg sonars record the heading and attitude sensor
134  * data streams input into the sonar, usually at a sampling
135  * frequency of 100 Hz.
136 \* 9. The beam angles and travel times are reported in the raw range
137  * and angle datagrams. The raw times and angles are recorded
138  * without correction for the vessel motion. The raw ranges are
139  * given as two-way travel times.
140  * 10. The sidescan is structured in terms of a certain number of samples
141  * per beam. The range sampling rate for the sidescan is the same as
142  * that specified in the depth datagram and the ranges in the sidescan
143  * datagram (seabed image datagram) are all two way travel times.
144  * 11. The attitude data is output asynchronously with respect to the
145  * ping output datagrams. Typical motion sensors give data at a 100 Hz
146  * rate. The attitude datagrams are given when the number of measurements
147  * is 100, or usually at 1 second intervals. The attitude data timing
148  * is corrected for the sensor time delay entered by the operator. If
149  * roll is input with respect to the horizontal, then the sonar
150  * recalculates the roll so that the output values are in the plane
151  * defined by the heading and pitch axis. The entered sensor offsets
152  * (roll bias, pitch bias, heading bias). Extra heave at the transducer
153  * due to roll and pitch when the sensor does not give its data at the
154  * transducer position is also included and heave is positive downwards.
155  * The sensor status is copied from the input datagram's two sync bytes
156  * with the second byte always set to 0x90. The first byte is either
157  * zero or in the 0x90-0xAF range. If the latter is true, then 0x90
158  * indicates valid data with full accuracy, 0x91-0x99 indicates valid
159  * data with gradually reduced accuracy, 0x9A-0x9F indicates invalid data
160  * from an operating sensor, and 0xA0-0xAF indicates invalid data
161  * from a faulty sensor. This interpretation may be dependent on the
162  * attitude sensor type.
163  * 12. The heading data is output asynchronously with respect to the
164  * ping output datagrams. Typical heading sensors give data at a 10 Hz
165  * rate. The attitude datagrams are given when the number of measurements
166  * is 100, or usually at 10 second intervals. The heading data is
167  * corrected for the heading offset entered by the operator.
168  * 13. The raw vendor format (format 58) does not support flagging of bathymetry
169  * values nor does it store navigation in the bathymetry data records.
170  * MB-System also supports a processing format (59) which includes
171  * beamflags and navigation in the bathymetry records. This format
172  * is identical to the vendor format except for the use of a
173  * slightly different bathymetry record.
174  * 14. The EM2040D outputs two simultaneous sets of datagrams for each
175  * receive head with the same ping number. The data can be distinguished
176  * only by the sonar serial number in each record.
177  *
178  */

468  given by the beam incidence angle adjustment (IBA). The beam
469  incidence angle re the horizontal, corrected for the ray bending,
470  can be calculated as follows:
471  BAC = atan( z / abs(y) ) + IBA.
472  BAC is positive downwards and IBA will be positive when the

483  0: Amplitude detect
484  1: Phase detect

487  detection. Bit 4-6 is used to specify how the range (and x,y,z
488  parameters) for this beam is calculated
489  0: Normal detection
490  1: Interpolated or extrapolated from neighbour detections
491  2: Estimated
492  3: Rejected candidate
493  4: No detection data is available for this beam (all parameters

495  5-7: Future use
