25  * mbf_mr1bldeo.h defines the data structures used by MBIO functions
26  * to store multibeam data read from the MBF_MR1BLDEO format (MBIO id 61).
27  *
28  * Author: D. W. Caress
29  * Date: October 23, 1995
30  *
31  *
32  */
33 /*
34  * Notes on the MBF_MR1BLDEO data format:
35  * 1. The MR1 post processing format uses the xdr external
36  * data representation for portability.
37  * 2. The data stream consists of a file header followed
38  * by individual pings.
39  * 3. The file header contains a comment string and the
40  * number of pings. The comment string is broken up
41  * into multiple comments by MBIO on reading; the comments
42  * are concatenated into a single string on writing.
43  * 4. The pings each contain a header plus the bathymetry and/or
44  * sidescan data.
45  * 6. The data structure defined below includes all of the values
46  * which are passed in the MR1 post processing format.
47  * 7. The data structure defined below also includes travel
48  * time values for each bathymetry beam - this is an
49  * addition to the HIG MR1 post processing format.
50  */
