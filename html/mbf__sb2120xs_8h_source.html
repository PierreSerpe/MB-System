35 /*
36  * Notes on the MBF_SB2120XS (XSE) data format:
37  * 1. SeaBeam Instruments introduced a new format called XSE in 1999.
38  * This "Data Exchange Format" is associated with the new
39  * version of "Hydrostar ONLINE" and represents the intended
40  * data format for both ELAC Bottomchart Compact MK II sonars
41  * (50 kHz and/or 180 kHz) and SeaBeam 2100 series sonars
42  * (12 kHz, 20 kHz, 36 kHz). This follows the purchase of
43  * SeaBeam Instruments by L3 Communications, the parent
44  * company of ELAC Nautik.
45  * 2. The XSE format implements a well defined binary format
46  * structure in which each data record is represented as
47  * a "frame" with the following structure:
48  * -------------------------------------------------------
49  * Item Bytes Format Value Units Description
50  * -------------------------------------------------------
51  * Start 4 ulong $HSF Frame start
52  * Byte Count 4 ulong bytes Between byte count
53  * and frame end
54  * Id 4 ulong Frame id - see below
55  * Source 4 ulong Sensor id
56  * Seconds 4 ulong seconds Seconds since
57  * 1/1/1901 00:00:00
58  * Microsec 4 ulong usec Microseconds
59  * ... ... ... ... ... Frame specific groups
60  * End 4 ulong #HSF Frame end
61  * -------------------------------------------------------
62  * Within each frame are "groups", each with the following structure:
63  * -------------------------------------------------------
64  * Item Bytes Format Value Units Description
65  * -------------------------------------------------------
66  * Start 4 ulong $HSG Group start
67  * Byte Count 4 ulong bytes Between byte count
68  * and group end
69  * Id 4 ulong Group id - see below
70  * ... ... ... ... ... Group specific data
71  * End 4 ulong #HSG Group end
72  * -------------------------------------------------------
73  * 2. The valid frames include:
74  * Frame Name Id Groups w/ group id's in ()
75  * ---------------------------------------------------------
76  * Navigation 1 General(1), Position(2),
77  * MotionGroundTruth(4), MotionThroughWater(5),
78  * CurrentTrack(6), HeaveRollPitch (7), Heave(8),
79  * Roll(9), Pitch(10), Heading(11), Log(12)
80  * Sidescan 5 General(1), Amplitude(4), Phase(5)
81  * Multibeam 6 General(1), Traveltime(3), Quality(4),
82  * Amplitude(5), Delay(6), Lateral(7),
83  * Along(8), Depth(9), Angle(10), Beam(1)
84  * Comment 99 General(1) **MB-System ONLY!!!!**
85  * 3. An additional set of SeaBeam 2100 specific frames are defined,
86  * but are not supported in this i/o module. Many other frames
87  * are defined, but not supported here. These are read and passed
88  * through MB-System as MB_DATA_OTHER type data records.
89  * 4. SeaBeam Instruments 2120 20KHz sonar systems output both bathymetry
90  * and amplitude information for up to 151 beams per multibeam frame.
91  * Each ping produces a variable number of beams.
92  * 5. The XSE format uses asynchronous navigation only; navigation
93  * is not included in the multibeam or sidescan pings.
94  * MB-System interpolates or extrapolates the available
95  * navigation as necessary.
96  *
97  */
