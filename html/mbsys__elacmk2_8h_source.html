24 /*
25  * mbsys_elacmk2.h defines the data structures used by MBIO functions
26  * to store data from Elac BottomChart Mark II multibeam sonar systems.
27  * The data formats which are commonly used to store Elac
28  * data in files include
29  * MBF_ELMK2UNB : MBIO ID 92
30  *
31  *
32  * Author: D. W. Caress (L-DEO)
33  * Date: August 20, 1994
34  *
35  */
36 /*
37  * Notes on the MBF_ELMK2UNB data format:
38  * 1. Elac multibeam systems output binary data telegrams.
39  * 2. Elac BottomChart Mark II sonar systems output both bathymetry
40  * and amplitude information for 126 beams per telegram.
41  * 3. Each ping produces 42 beams. A wide swath is constructed
42  * by successively pinging in different directions.
43  * 4. Each telegram is preceded by a two byte start code and
44  * followed by a three byte end code consisting of 0x03
45  * followed by two bytes representing the checksum for
46  * the data bytes. MB-System does not calculate checksums
47  * and puts 0's in the checksum bytes.
48  * 5. The relevant telegram start codes, types, and sizes are:
49  * 0x0250: Comment (Defined only for MB-System) 200 data bytes
50  * 0x0251: Position 36 data bytes
51  * 0x0252: Parameter 54 data bytes
52  * 0x0253: Sound velocity profile 2016 data bytes
53  * 0x0258: Mark II general bathymetry wrapper 24 data bytes
54  * Mark II general bathymetry beam 28 data bytes
55  * 6. Elac systems record navigation fixes using the position
56  * telegram; navigation is not included in the per ping data.
57  * Since speed is not recorded, it is impossible to extrapolate
58  * position from the last navigation fix when processing the
59  * data serially, as MBIO does. It may thus be necessary to extract
60  * the navigation from the position telegrams and remerge it with
61  * the ping telegrams using the program mbmerge.
62  *
63  */
