25  * mbsys_sb2100.h defines the data structures used by MBIO functions
26  * to store data from SeaBeam 2100 and 1000 multibeam sonar systems.
27  * The data formats which are commonly used to store SeaBeam 1000/2100
28  * data in files include
29  * MBF_SB2100RW : MBIO ID 41
30  * MBF_SB2100BN : MBIO ID 42

32  * Author: D. W. Caress

36 /*
37  * Notes on the MBSYS_SB2100 data structure:
38  * 1. SeaBeam 2100 multibeam sonars output raw data in two
39  * formats, with a third created in post processing.
40  * The older format is mostly ascii with binary
41  * sidescan (format 41). The newer formats are entirely
42  * binary. Format 42 contains all information; format 43
43  * is identical to format 42 except that the sidescan
44  * data has been removed.
45  * 2. The SeaBeam 2100 sonars output up to 151 beams of
46  * bathymetry and 2000 pixels of sidescan, along with a
47  * plethora of other information.
48  * 3. The records all include navigation and time stamp information.
49  * The record types are for format 41 are:
50  * PR: sonar parameter record (roll bias, pitch bias, sound velocity profile)
51  * TR: sonar text record (comments)
52  * DR: bathymetry data record (bathymetry and per-beam amplitudes)
53  * SS: side scan data record
54  * The record types are for format 42 are:
55  * PR: sonar parameter record (roll bias, pitch bias, sound velocity profile)
56  * TR: sonar text record (comments)
57  * DH: ping data header (one per ping)
58  * BR: bathymetry data record (bathymetry and per-beam amplitudes)
59  * SR: side scan data record
60  * 4. For format 41 a single ping usually results in both DR and SS
61  * records. The PR record occurs every 30 minutes or when
62  * the sound velocity profile or bias parameters are changed.
63  * 5. For format 42 a single ping results in DH, BR, and SR records.
64  * Format 42 files created directly by the sonars will have
65  * PR records at the beginning; format 42 files created by
66  * translating format 41 files will have the same frequency of
67  * PR records as in the original files.
68  * 5. The kind value in the mbsys_sb2k_struct indicates whether the
69  * mbsys_sb2k_data_struct structure holds data from a ping or
70  * data from some other record:
71  * kind = 1 : data from a ping
72  * (DR + SS) or (DH + BR + SR)
73  * kind = 2 : comment (TR)
74  * kind = 8 : sonar parameter (PR)
75  * 6. The data structure defined below includes all of the values
76  * which are passed in SeaBeam 2100 records.
77  */
78 /*
79  * Notes on the MBF_SB2100B1 data format:
80  * 1. SeaBeam 2100 multibeam sonars currently generate
81  * raw data in an hybrid ascii/binary format (41).
82  * This is a replacement fully binary (excepting file header)
83  * format which has significantly faster i/o during processing.
84  * 2. The SeaBeam 2100 sonars output up to 151 beams of bathymetry
85  * and 2000 pixels of sidescan measurements, along with a plethora
86  * of other information.
87  * 3. The record types are:
88  * SB21BIFH: file header with data format description
89  * (beginning of file only)
90  * SB21BIPR: sonar parameter record
91  * (roll bias, pitch bias, sound velocity profile)
92  * SB21BITR: sonar text record (comments)
93  * SB21BIDH: sonar data header (one for each ping)
94  * SB21BIBR: bathymetry data record (one for each ping)
95  * SB21BISR: sidescan data record (one for each ping)
96  * 4. The file header record occurs at the beginning of each file.
97  * this is a fully ASCII record with line feeds and null
98  * termination so that uninformed users can figure out the
99  * contents of the file without additional documentation.
100  * There is no analog to this header in format 41.
101  * 5. The parameter record should be generated at the beginning of
102  * every file (after the header); new files with new parameter
103  * records should be generated any time the roll bias, pitch
104  * bias, or sound velocity profile values change.
105  * The existing SeaBeam 2100 sonars output parameter records when
106  * the sonar begins logging and every 30 minutes thereafter,
107  * regardless of where it appears in files.
108  * The parameter also includes values for navigation offsets
109  * due to the offset between the transducers and the GPS antenna.
110  * SeaBeam sonars do not presently make use of such parameters.
111  * 6. Individual comment records are limited to lengths of
112  * 1920 characters.
113  * Each file should begin with comment records stating
114  * the sonar and sonar control software version used to
115  * generate the data. This does not occur at present.
116  * 7. Each ping generates three data records in the following
117  * order:
118  * SB21BIDH: sonar data header
119  * SB21BIBR: bathymetry data record
120  * SB21BISR: sidescan data record
121  * 8. The data structure defined below includes all of the values
122  * which are passed in SeaBeam 2100 records.
123  *
124  *
125  */
126 /*
127  * Notes on the MBF_SB2100B2 data format:
128  * 1. SeaBeam 2100 multibeam sonars currently generate
129  * raw data in an hybrid ascii/binary format (41).
130  * Format 42 is a replacement fully binary (excepting file header)
131  * format which has significantly faster i/o during processing.
132  * This format is a bathymetry-only variant of format 42 which,
133  * due to the lack of sidescan data, has much smaller data files
134  * and much faster i/o.
135  * 2. The SeaBeam 2100 sonars output up to 151 beams of bathymetry
136  * and 2000 pixels of sidescan measurements, along with a plethora
137  * of other information.
138  * 3. The record types are:
139  * SB21BIFH: file header with data format description
140  * (beginning of file only)
141  * SB21BIPR: sonar parameter record
142  * (roll bias, pitch bias, sound velocity profile)
143  * SB21BITR: sonar text record (comments)
144  * SB21BIDH: sonar data header (one for each ping)
145  * SB21BIBR: bathymetry data record (one for each ping)
146  * SB21BISR: sidescan data record (one for each ping)
147  * 4. The file header record occurs at the beginning of each file.
148  * this is a fully ASCII record with line feeds and null
149  * termination so that uninformed users can figure out the
150  * contents of the file without additional documentation.
151  * There is no analog to this header in format 41.
152  * 5. The parameter record should be generated at the beginning of
153  * every file (after the header); new files with new parameter
154  * records should be generated any time the roll bias, pitch
155  * bias, or sound velocity profile values change.
156  * The existing SeaBeam 2100 sonars output parameter records when
157  * the sonar begins logging and every 30 minutes thereafter,
158  * regardless of where it appears in files.
159  * The parameter also includes values for navigation offsets
160  * due to the offset between the transducers and the GPS antenna.
161  * SeaBeam sonars do not presently make use of such parameters.
162  * 6. Individual comment records are limited to lengths of
163  * 1920 characters.
164  * Each file should begin with comment records stating
165  * the sonar and sonar control software version used to
166  * generate the data. This does not occur at present.
167  * 7. Each ping generates three data records in the following
168  * order:
169  * SB21BIDH: sonar data header
170  * SB21BIBR: bathymetry data record
171  * SB21BISR: sidescan data record
172  * The sidescan records are present in format 43, but contain
173  * no pixels. This allows format 43 files to be read as
174  * format 42 without breaking anything.
175  * 8. The data structure defined below includes all of the values
176  * which are passed in SeaBeam 2100 records.
177  *
178  */
