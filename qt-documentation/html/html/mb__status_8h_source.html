384 /*
385  * The following defines the values used to flag or
386  * select individual bathymetry values (soundings). This scheme
387  * is very similar to the convention used in the HMPS
388  * hydrographic data processing package and the SAIC Hydrobat
389  * package. The values passed in MBIO functions are single
390  * byte characters.
391  *
392  * Macros used to identify the flags are also defined here.
393  *
394  * The flagging scheme is as follows:
395  *
396  * Beams cannot be both flagged and selected. However, more than
397  * one "reason bit" can be set for either flagging or selection.
398  *
399  * The flag and select bits:
400  * xxxxxx00 => This beam is neither flagged nor selected.
401  * xxxxxx01 => This beam is flagged as bad and should be ignored.
402  * xxxxxx10 => This beam has been selected.
403  *
404  * Flagging modes:
405  * 00000001 => Flagged because no detection was made by the sonar.
406  * xxxxx101 => Flagged by manual editing.
407  * xxxx1x01 => Flagged by automatic filter.
408  * xxx1xx01 => Flagged by automatic filter in the current program
409  * xx1xxx01 => Flagged because this is a secondary bottom pick.
410  * x1xxxx01 => Flagged because this is an interpolated rather than observed sounding
411  * 1xxxxx01 => Flagged as unreliable using original quality values from sonar.
412  *
413  * xxx1xx01 => Flagged because uncertainty exceeds 1 X IHO standard. (original meaning, deprecated)
414  * xx1xxx01 => Flagged because uncertainty exceeds 2 X IHO standard. (original meaning, deprecated)
415  * x1xxxx01 => Flagged because footprint is too large (original meaning, deprecated)
416  *
417  * Selection modes:
418  * 00000010 => Selected, no reason specified.
419  * xxxxx110 => Selected as least depth.
420  * xxxx1x10 => Selected as average depth.
421  * xxx1xx10 => Selected as maximum depth.
422  * xx1xxx10 => Selected as location of sidescan contact.
423  * x1xxxx10 => Selected, spare.
424  * 1xxxxx10 => Selected, spare.
425  *
426  */
