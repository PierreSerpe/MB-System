24 /*
25  * mbsys_dsl.h defines the data structures used by MBIO functions
26  * to store data from the DSL AMS-120 sonar system. This is a 120 khz
27  * deep-towed sonar which produces both sidescan and bathymetry.
28  * The data formats used to store the DSL AMS-120 data are:
29  * MBF_DSL120PF : MBIO ID 111
30  * MBF_DSL120SF : MBIO ID 112
31  *
32  *
33  * Author: D. W. Caress
34  * Date: August 5, 1996
35  *
36  *
37  */
38 /*
39  * Notes on the MBSYS_DSL data structure:
40  * 1. The DSL processing system used a parallel file scheme
41  * in which bathymetry, sidescan, and navigation are kept
42  * in separate files. Some examples of the filenames are:
43  * DSL120.940630_1100.bat.dat - bathymetry
44  * DSL120.940630_1100.amp.dat - sidescan
45  * DSL120.940630_1100.nav - navigation
46  * 2. The DSL parallel file scheme is supported under MB-System
47  * data format 111 (MBF_DSL120PF); a single file scheme is
48  * supported under data format 112 (MBF_DSL120SF). The
49  * single file scheme is within the DSL format specification.
50  * 3. The bathymetry and sidescan data are stored in binary
51  * data structures; the navigation is stored in ASCII.
52  * 4. The DSL format supports arbitrary numbers of bathymetry
53  * and sidescan values. The MB-System implementation has
54  * maximum numbers of values hardwired in the #defines
55  * below.
56  * 5. The bathymetry and sidescan data have navigation
57  * fields, but these navigation values typically repeat
58  * for many pings, often being the same for entire files.
59  * The separate navigation files contain the post-processed
60  * navigation with distinct values for each ping.
61  * 6. The original navigation values supplied by WHOI/DSL
62  * so far are in projected eastings and northings rather
63  * than in longitude and latitude. Since MB-System only
64  * works with longitude and latitude, a special program
65  * will need to be used to translate the navigation to
66  * longitude and latitude.
67  * 7. Due to the above problem, the following scheme will
68  * be used to handle MBF_DSL120DT data in MB-System:
69  * a) Run a perl macro called mbdslnavfix on the
70  * DSL120.*.nav files, producing DSL.*.mbnav
71  * files in a format understood by mbmerge.
72  * b) Run mbmerge on the DSL120.*.bat.dat and DSL120.*.amp.dat
73  * files to merge the navigation from the DSL120.*.mbnav
74  * in with the bathymetry and sidescan data.
75  * 8. The parallel file structure is difficult to deal with
76  * in the MB-System world where programs expect a single
77  * input and a single output file name. Handling the two
78  * files (bat.dat and amp.dat) will be handled in low level
79  * i/o routines. If the input file name has a "bat" in it,
80  * the code will attempt to open a second file with the
81  * same name except that "amp" is substituted for "bat".
82  * If the specified input file has "amp" in it, then the
83  * second file will have "bat". The same will be true for
84  * the output files.
85  * 9. The comment records reside only in the "bat" files in
86  * the parallel file scheme of format 111.
87  * The comment records are an MB-System extension to the
88  * DSL format. The program mbcopy can be used to strip
89  * the comments out of the data files prior to reusing
90  * DSL processing software.
91  *
92  */
