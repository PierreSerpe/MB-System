33 /*
34  * Notes on the MBSYS_HDCS data structure:
35  * 1. The OMG-HDCS format is a collection of similar data
36  * formats used by the seafloor mapping software
37  * developed by Dr. John Hughes Clarke (Ocean Mapping
38  * Group of the University of New Brunswick). A variety
39  * of sonars are supported in OMG-HDCS.
40  * 2. OMG-HDCS files all begin with a summary header that
41  * specifies the format version, the data source
42  * (type of sonar), the number of records, and the
43  * minimum and maximum values of position and data
44  * values.
45  * 3. The summary header is followed by a set of uniformly
46  * sized data records. The data record size is determined
47  * by the format version and data source. Ecch record is
48  * is divided into a profile (header) and an array of
49  * beam structures.
50  * 4. Sidescan imagery can be stored in parallel files in
51  * the same directory as the primary bathymetry and
52  * amplitude files. The sidescan files have the suffix
53  * ".ss_data" added to the end of the primary file's name.
54  * 5. Comment records are encoded in MBIO by setting the first
55  * eight bytes of the data record header (profile) to '#'
56  * values. The profile part of the comment record will have
57  * the same size as expected for data from the tool type
58  * listed in the summary. However, the data section for
59  * comment records will always be 256 bytes long regardless
60  * of the tool type. This mechanism is supported only by
61  * MB-System.
62  *
63  */
