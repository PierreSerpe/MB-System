24 /*
25  * mbsys_surf.h defines the MBIO data structures for handling data from
26  * SAM Electronics multibeam sonars in the Atlas processing format SURF.
27  * The relevant sonars include Hydrosweep DS2, MD2 and Fansweep sonars.
28  * The older Hydrosweep DS and MD sonars produce data in different
29  * formats (e.g. 21-24 and 101-102).
30  * The data format associated with the SURF format is:
31  * MBSYS_SURF formats (code in mbsys_surf.c and mbsys_surf.h):
32  * MBF_SAMESURF : MBIO ID 181 - Vendor processing format
33  *
34  *
35  * Author: D. W. Caress
36  * Author: D. N. Chayes
37  * Date: June 13, 2002
38  *
39  *
40  */
41 /*
42  * Notes on the MBSYS_SURF data structure:
43  *
44  * 1) STN Atlas Marine Electronics (aka SAM) sonars write raw data in real-time
45  * as binary XDR encoded data. Files are stored on disk by the HYDROMAP
46  * Online workstation. The workstation on the Ewing is an HP Vectra
47  * running SuSe Linux (2.2 kernel.)
48  *
49  * 2) The HYDROMAP Offline software translates the data into a processing
50  * format called SURF
51  *
52  * 3) Multiple parallel files are created. For example:
53  * The '.six' file contains global data and reference tables
54  * The '.sda' file contains sounding dependent mass data
55  *
56  * 4) SAM provides an open source library (SAPI) to read and write
57  * SURF data
58  */
