25  * mbsys_xse.h defines the data structures used by MBIO functions
26  * to store swath sonar data in the XSE Data Exchange Format
27  * developed by L-3 Communications ELAC Nautik.
28  * This format is used for data from ELAC Bottomchart multibeam sonars
29  * and SeaBeam 2100 multibeam sonars (made by L-3 Communications
30  * SeaBeam Instruments).
31  * The data format associated with XSE is:
32  * MBF_L3XSERAW : MBIO ID 94
33  *
34  *
35  * Author: D. W. Caress
36  * Date: August 1, 1999
37  * Additional Authors: P. A. Cohen and S. Dzurenko
38  *
39  *
40  *

42 /*
43  * Notes on the MBSYS_XSE (XSE) data format:
44  * 1. L3 Communications introduced a new format called XSE in 1999.
45  * SeaBeam Instruments (maker of SeaBeam multibeam sonars)
46  * and Elac Nautik (make of Bottomchart multibeam sonars)
47  * are both divisions of L3 Communications.
48  * This "Data Exchange Format" is associated with the new
49  * version of "Hydrostar ONLINE" and represents the intended
50  * data format for both ELAC Bottomchart Compact MK II sonars
51  * (50 kHz and/or 180 kHz) and SeaBeam 2100 series sonars
52  * (12 kHz, 20 kHz, 36 kHz). This follows the purchase of
53  * SeaBeam Instruments by L3 Communications, the parent
54  * company of ELAC Nautik.
55  * 2. The XSE format implements a well defined binary format
56  * structure in which each data record is represented as
57  * a "frame" with the following structure:
58  * -------------------------------------------------------
59  * Item Bytes Format Value Units Description
60  * -------------------------------------------------------
61  * Start 4 ulong $HSF Frame start
62  * Byte Count 4 ulong bytes Between byte count
63  * and frame end
64  * Id 4 ulong Frame id - see below
65  * Source 4 ulong Sensor id
66  * Seconds 4 ulong seconds Seconds since
67  * 1/1/1901 00:00:00
68  * Microsec 4 ulong usec Microseconds
69  * ... ... ... ... ... Frame specific groups
70  * End 4 ulong #HSF Frame end
71  * -------------------------------------------------------
72  * Within each frame are "groups", each with the following structure:
73  * -------------------------------------------------------
74  * Item Bytes Format Value Units Description
75  * -------------------------------------------------------
76  * Start 4 ulong $HSG Group start
77  * Byte Count 4 ulong bytes Between byte count
78  * and group end
79  * Id 4 ulong Group id - see below
80  * ... ... ... ... ... Group specific data
81  * End 4 ulong #HSG Group end
82  * -------------------------------------------------------
83  * 2. The valid frames include:
84  * Frame Name Id Groups w/ group id's in ()
85  * ---------------------------------------------------------
86  * Navigation 1 General(1), Position(2), Accuracy(3),
87  * MotionGroundTruth(4), MotionThroughWater(5),
88  * CurrentTrack(6), HeaveRollPitch(7), Heave(8),
89  * Roll(9), Pitch(10), Heading(11), Log(12),
90  * GPS(13)
91  * Sound Velocity 2 General(1),
92  * Depth(2), Velocity(3),
93  * Conductivity(4), Salinity (5), Temperature(6),
94  * Pressure(7), SSV(8), Position(9)
95  * Tide 3 General(1), Position(2),
96  * Time(3), Tide(4)
97  * Ship 4 General(1), Attitude(2),
98  * Position(3), Dynamics(4), Motion(5),
99  * Geometry(6), Description(7),
100  * Parameter(8), NavigationAndMotion(9),
101  * Transducer(10), TransducerExtended(11)
102  * Sidescan 5 General(1), AmplitudeVsTravelTime(2),
103  * PhaseVsTravelTime(3),
104  * Amplitude(4), Phase(5), Signal(6), PingType(7)
105  * ComplexSignal(8), Weighting(9)
106  * Multibeam 6 General(1), Beam(2),
107  * Traveltime(3), Quality(4),
108  * Amplitude(5), Delay(6), Lateral(7),
109  * Along(8), Depth(9), Angle(10),
110  * Heave(11), Roll(12), Pitch(13),
111  * Gates(14), Noise(15), EchoLength(16),
112  * Hits(17), HeaveReceive(18), Azimuth(19),
113  * MBsystemNavigation(99)
114  * Single Beam 7 General(1)
115  * Control 8 Request(1), Insert(2), Change(3),
116  * Add(4), Delete(5), Action(6), Reply(7)
117  * Bathymetry 9 General(1), Position(2), Depth(3)
118  * Project 10 General(1), Server(2), Status(3), Sources(4)
119  * Native 11 General(1), RWCollectable(2), UNB(3),
120  * Raw(4), ELAC(5)
121  * Geodetic 12 General(1), Ellipsoid(2), Datum(3),
122  * Projection(4), System(5), Alias(6)
123  * SeaBeam 13 Properties(1), HeaveRollPitch(2), Setup(3),
124  * MotionReferenceUnit(4), Settings(5),
125  * Beams(6), Gates(7), Raw(8), Center(9),
126  * Sidescan, Shutdown(), Ping(), Calibrate(),
127  * Collect(), Surface(), Hydrophone(),
128  * Projector(17), Calibration(18), Acknowledge(19),
129  * Warning(20), Message(21), Error(22)
130  * Comment 99 General(1) **MB-System ONLY!!!!**
131  * 3. Not all of these frames are directly supported by this MB-System
132  * i/o module. Unsupported frames are read and passed
133  * through MB-System as MB_DATA_OTHER type data records.
134  * 4. SeaBeam Instruments 2120 12 kHz and 20 KHz multibeam sonar
135  * systems output both bathymetry and amplitude
136  * information for up to 151 beams per multibeam frame.
137  * Each ping produces a variable number of beams.
138  * 5. Elac Bottomchart MkII 50 KHz and 180 kHz sonar systems
139  * output both bathymetry and amplitude information
140  * for up to 126 beams per multibeam frame.
141  * Each ping produces a variable number of beams.
142  * 6. The raw XSE format provides asynchronous navigation only; navigation
143  * is not included in the multibeam or sidescan pings.
144  * MB-System adds MBsystemNavigation(99) groups to the multibeam
145  * frames so that processed navigation can be directly associated
146  * with each survey ping..
147  * 7. The comment records are supported by MB-System only and are
148  * not part of the L3 Communications XSE format specification.
149  * The comment frame encloses a single general comment group (id=1).
150  * -------------------------------------------------------
151  * Item Bytes Format Value Units Description
152  * -------------------------------------------------------
153  * Start 4 ulong $HSF Frame start
154  * Byte Count 4 ulong bytes Between byte count
155  * and frame end
156  * Id 4 ulong 99 Frame id - see below
157  * Source 4 ulong 0 Sensor id
158  * Seconds 4 ulong seconds Seconds since
159  * 1/1/1901 00:00:00
160  * Microsec 4 ulong usec Microseconds
161  * ... ... ... ... ... Frame specific groups
162  * End 4 ulong #HSF Frame end
163  * -------------------------------------------------------
164  * The only comment group is 1:
165  * -------------------------------------------------------
166  * Item Bytes Format Value Units Description
167  * -------------------------------------------------------
168  * Start 4 ulong $HSG Group start
169  * Byte Count 4 ulong bytes Between byte count
170  * and group end
171  * Id 4 ulong 1 Group id - see below
172  * N 4 long bytes Length of null terminated
173  * comment string, padded
174  * to a multiple of 4
175  * Comment N char chars Comment string
176  * End 4 ulong #HSG Group end
177  * -------------------------------------------------------
178  *
179  */
