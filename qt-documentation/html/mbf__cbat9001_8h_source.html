25  * mbf_cbat9001.h defines the data structures used by MBIO functions
26  * to store multibeam data read from the MBF_CBAT9001 format (MBIO id 81).
27  *
28  * Author: D. W. Caress
29  * Date: August 21, 1994
30  *
31  */
32 /*
33  * Notes on the MBF_CBAT9001 data format:
34  * 1. Reson SeaBat products are high frequency,
35  * shallow water multibeam sonars.
36  * Reson SeaBat 9001 systems output both bathymetry
37  * and amplitude information for 60 beams.
38  * Reson SeaBat 8101 systems output both bathymetry
39  * and amplitude information for up to 128 beams.
40  * These sonars use fixed, analog beamforming followed
41  * by a combination of amplitude and phase bottom
42  * detection.
43  * 2. Reson multibeam systems output raw range and amplitude
44  * data in a binary format. The data acquisition systems
45  * associated with the sonars calculate bathymetry using
46  * a water sound velocity, roll, pitch, and heave data.
47  * 3. Generally, Reson data acquisition systems record
48  * navigation asynchronously in the data stream, without
49  * providing speed information. This means that the
50  * navigation must be interpolated on the fly as the
51  * data is read.
52  * 4. The navigation is frequently provided in projected
53  * coordinates (eastings and northings) rather than in
54  * longitude and latitude. Since MB-System operates solely
55  * in longitude and latitude, the original navigation must
56  * be unprojected.
57  * 5. The Reson data formats supported by MB-System include:
58  * MBF_CBAT9001 - a binary format designed by John Hughes Clarke
59  * of the University of New Brunswick. Parameter and
60  * sound velocity profile records are included.
61  * MBF_CBAT8101 - a clone of the above format supporting
62  * Reson 8101 data.
63  * MBF_HYPC8101 - the ASCII format used by the HYPACK system
64  * of Coastal Oceanographics in conjunction with
65  * Reson 8101 data. This format is supported as read-only
66  * by MB-System.
67  * MBF_GSFGENMB - the generic sensor format of SAIC which
68  * supports data from a large number of sonars, including
69  * Reson sonars. MB-System handles GSF separately from
70  * other formats.
71  * 6. For the UNB-style formats MBF_CBAT9001 and MBF_CBAT8101,
72  * each data telegram is preceded by a two byte start code and
73  * followed by a three byte end code consisting of 0x03
74  * followed by two bytes representing the checksum for
75  * the data bytes. MB-System does not calculate checksums
76  * and puts 0's in the checksum bytes.
77  * The relevant telegram start codes, types, and sizes are:
78  * 0x0240: Comment*** 200 data bytes
79  * 0x0241: Position 36 data bytes
80  * 0x0242: Parameter 44 data bytes
81  * 0x0243: Sound velocity profile 2016 data bytes
82  * 0x0244: SeaBat 9001 bathymetry 752 data bytes
83  * 0x0245: Short sound velocity profile 816 data bytes
84  * 0x0246: SeaBat 8101 bathymetry*** 1244 data bytes
85  * 0x0247: Heading*** 752 data bytes
86  * 0x0248: Attitude*** 752 data bytes
87  * *** Defined only for MB-System
88  *
89  */
