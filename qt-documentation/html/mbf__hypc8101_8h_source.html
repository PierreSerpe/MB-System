33 /*
34  * Notes on the MBF_HYPC8101 data format:
35  * 1. Reson SeaBat products are high frequency,
36  * shallow water multibeam sonars.
37  * Reson SeaBat 9001 systems output both bathymetry
38  * and amplitude information for 60 beams.
39  * Reson SeaBat 8101 systems output both bathymetry
40  * and amplitude information for up to 101 beams.
41  * These sonars use fixed, analog beamforming followed
42  * by a combination of amplitude and phase bottom
43  * detection.
44  * 2. Reson multibeam systems output raw range and amplitude
45  * data in a binary format. The data acquisition systems
46  * associated with the sonars calculate bathymetry using
47  * a water sound velocity, roll, pitch, and heave data.
48  * 3. Generally, Reson data acquisition systems record
49  * navigation asynchronously in the data stream, without
50  * providing speed information. This means that the
51  * navigation must be interpolated on the fly as the
52  * data is read.
53  * 4. The navigation is frequently provided in projected
54  * coordinates (eastings and northings) rather than in
55  * longitude and latitude. Since MB-System operates solely
56  * in longitude and latitude, the original navigation must
57  * be unprojected.
58  * 5. The Reson data formats supported by MB-System include:
59  * MBF_HYPC8101 - a binary format designed by John Hughes Clarke
60  * of the University of New Brunswick. Parameter and
61  * sound velocity profile records are included.
62  * MBF_CBAT8101 - a clone of the above format supporting
63  * Reson 8101 data.
64  * MBF_HYPC8101 - the ASCII format used by the HYPACK system
65  * of Coastal Oceanographics in conjunction with
66  * Reson 8101 data. This format is supported as read-only
67  * by MB-System.
68  * MBF_GSFGENMB - the generic sensor format of SAIC which
69  * supports data from a large number of sonars, including
70  * Reson sonars. MB-System handles GSF separately from
71  * other formats.
72  * 6. For the UNB-style formats MBF_HYPC8101 and MBF_CBAT8101,
73  * each data telegram is preceded by a two byte start code and
74  * followed by a three byte end code consisting of 0x03
75  * followed by two bytes representing the checksum for
76  * the data bytes. MB-System does not calculate checksums
77  * and puts 0's in the checksum bytes.
78  * The relevant telegram start codes, types, and sizes are:
79  * 0x0240: Comment*** 200 data bytes
80  * 0x0241: Position 36 data bytes
81  * 0x0242: Parameter 44 data bytes
82  * 0x0243: Sound velocity profile 2016 data bytes
83  * 0x0244: SeaBat 9001 bathymetry 752 data bytes
84  * 0x0245: Short sound velocity profile 816 data bytes
85  * 0x0246: SeaBat 8101 bathymetry*** 1244 data bytes
86  * 0x0247: Heading*** 752 data bytes
87  * 0x0248: Attitude*** 752 data bytes
88  * *** Defined only for MB-System
89  * 7. For the ASCII HYPACK format, the following information
90  * is take verbatim from Coastal Oceanographics documentation:
91  *
92  * Coastal Oceanographics, Inc.
93  * Technical Note: Hypack Raw Data Format
94  *
95  * Data collected by the Hypack Survey program is recorded
96  * in Raw format, one file per survey line. Raw files are
97  * recorded as text, allowing them to be loaded into any
98  * text editor that reads large files (Windows Notepad, Write
99  * and Wordpad for example).
100  *
101  * When inspecting raw files, one of the first things noticed
102  * is that the format is not tabular. That is, there is not a record for
103  * each sounding containing depth, position, tide corrections,
104  * etc. Instead, there are separate records for each device
105  * measurement and the correlation between measurements is
106  * through time tags.
107  *
108  * Every raw file contains two section; a header, which is
109  * written when data logging starts, and a data section, which is
110  * written as data is collected. Each record starts with a
111  * three character tag. The tags are:
112  *
113  * Header
114  * ------
115  * DEV - Device Information
116  * EOH - End of Header
117  * EOL - End of Planned Line
118  * FTP - File Type
119  * INF - General Information
120  * LBP - Planned Line Begin Point
121  * LIN - Planned Line Data follows
122  * LNN - Planned Line Name
123  * OFF - Device Offsets
124  * PRD - Private Device Data
125  * PRI - Primary Navigation Device
126  * PTS - Planned Line Waypoint
127  * TND - Survey Time and Date
128  *
129  * Data
130  * ----
131  * FIX - Fix (Event) Mark
132  * HCP - Heave Compensation
133  * EC1 - Echo Sounding (single frequency)
134  * EC2 - Echo Sounding (dual frequency)
135  * ECM - Echo Soundings (multiple transducer system)
136  * GYR - Gyro Data (Heading)
137  * POS - Position
138  * ROX - Roxann data
139  * SB2 - Multibeam data
140  *
141  * --------------
142  * Header Section
143  * --------------
144  *
145  * DEV - Device Information
146  * ------------------------
147  * DEV dn dc "Device Name"
148  * dn: Device Number
149  * dc: Device Capabilities is a bit coded field. The bit definitions are:
150  * Bit Mask Meaning
151  * 0 1 Device provides Range/Range positions
152  * 1 2 Device provides Range/Azimuth positions
153  * 2 4 Device provides Lat/Long (e.g. GPS)
154  * 3 8 Device provides grid positions XY
155  * 4 16 Device provides echo soundings
156  * 5 32 Device provides heading
157  * 6 64 Device provides ship speed
158  * 7 128 Hypack clock is synched to device clock
159  * 8 256 Device provides tides
160  * 9 512 Device provides heave, pitch and roll
161  * 10 1024 Device is an ROV
162  * 11 2048 Device is a Left/Right Indicator
163  * 12 4096 Device accepts annotation strings
164  * 13 8192 Device accepts output from Hypack
165  * 14 16384 xxx
166  * 15 32768 Device has extended capabilities
167  *
168  * Example:
169  * DEV 0 100 "GPS"
170  *
171  * INF - General Information
172  * -------------------------
173  * INF "surveyor" "boat" "project" "area" tc dc sv
174  * tc: initial tide correction
175  * dc: initial draft correction
176  * sv: sound velocity
177  *
178  * Example:
179  * INF "steve" "LCH 19" "mcmillen" "617.6 to 618.2" -0.7 0 1500.0
180  *
181  * EOH - End of Header
182  * -------------------
183  * This tag simply indicates end of header and has no data.
184  *
185  * EOL - End of Planned Line
186  * -------------------------
187  * This tag simply indicates end of planned line information no data.
188  *
189  * LBP - Planned Line Begin Point.
190  * -------------------------------
191  * LBP x y
192  * x: x grid position
193  * y: y grid position
194  *
195  * Example:
196  * LBP 5567222.42 3771640.72
197  *
198  * LIN - Planned Line Data follows
199  * -------------------------------
200  * LIN nw
201  * nw: Number of waypoints
202  *
203  * Example:
204  * LIN 5
205  *
206  * LNN - Planned Line Name
207  * -----------------------
208  * LNN text
209  * text: line name or number
210  *
211  * Example:
212  * LNN 14
213  *
214  * OFF - Device Offsets
215  * --------------------
216  * OFF dn n1 n2 n3 n4 n5 n6 n7
217  * dn: device number
218  * n1: starboard, port offset. Positive starboard.
219  * n2: forward, aft offset. Positive forward
220  * n3: height (antenna) or depth (transducer draft) offset. Always positive.
221  * n4: yaw rotation angle. Positive for clockwise rotation.
222  * n5: roll rotation angle. Port side up is positive.
223  * n6: pitch rotation angle. Bow up is positive.
224  * n7: device latency in seconds.
225  *
226  * Example:
227  * OFF 0 0 0 13.35 0 0 0 0.86
228  *
229  * PRD - Private Device Data
230  * -------------------------
231  * PRD - Multiple transducer offset
232  * PRD dn OFF n1 n2 n3
233  * dn: device number
234  * n1: transducer starboard offset
235  * n2: transducer forward offset
236  * n3: transducer depth offset (draft)
237  *
238  * Example:
239  * PRD 1 OFF -25.60 0.00 0.40
240  *
241  * PRD - Odom Echoscan II Multibeam Identifier
242  * -------------------------------------------
243  * PRD dn ECHOSCN2 n1 n2
244  * dn: device number
245  * n1: Not used
246  * n2: Beam width
247  *
248  * Example:
249  * PRD 1 ECHOSCN2 -43.5 3.0
250  *
251  * PRD - Reson Seabat 9001 Multibeam Identifier
252  * --------------------------------------------
253  * PRD dn SEABAT n1 n2
254  * dn: device number
255  * n1: Not used
256  * n2: Beam width
257  *
258  * Example:
259  * PRD 1 SEABAT -44.2 1.5
260  *
261  * PRD - Reson Seabat 9003 Multibeam Identifier
262  * --------------------------------------------
263  * PRD dn SEA9003 n1 n2
264  * dn: device number
265  * n1: Not used
266  * n2: Beam width
267  *
268  * Example:
269  * PRD 1 SEA9003 -60.0 3.0
270  *
271  * PRD - Reson Seabat 8101 Multibeam Identifier
272  * --------------------------------------------
273  * PRD dn SEA8101 n1 n2 n3
274  * dn: device number
275  * n1: Beam 1 angle
276  * n2: Angle increment
277  * n3: Number of beams
278  *
279  * Example:
280  * PRD 1 SEA8101 -75.00 1.50 101
281  *
282  * PRI - Primary Navigation Device
283  * -------------------------------
284  * PRI dn
285  * dn: device number
286  *
287  * Example:
288  * PRI 0
289  *
290  * PTS - Planned Line Waypoint
291  * ---------------------------
292  * PTS x y
293  * x: waypoint easting
294  * y: waypoint northing
295  *
296  * Example:
297  * PTS 5569134.63 3774182.61
298  *
299  * TND - Survey Time and Date
300  * --------------------------
301  * TND t d
302  * t: time string
303  * d: date string
304  *
305  * Example:
306  * TND 15:54:33 08/28/95
307  *
308  * --------------------------
309  * Data Section
310  * --------------------------
311  *
312  * FIX - Fix (Event) Mark
313  * ----------------------
314  * FIX n
315  * n: event number
316  *
317  * Example:
318  * FIX 152
319  *
320  * HCP - Heave Compensation
321  * ------------------------
322  * HCP dn t h r p
323  * dn: device number
324  * t: time tag (seconds past midnight)
325  * h: heave in meters
326  * r: roll in degrees (+ port side up)
327  * p: pitch in degrees (+ bow up)
328  *
329  * Example:
330  * HCP 2 57273.81 0 3.61 0
331  *
332  * EC1 - Echo Sounding (single frequency)
333  * --------------------------------------
334  * EC1 dn t rd
335  * dn: device number
336  * t: time tag (seconds past midnight)
337  * rd: raw depth
338  *
339  * Example:
340  * EC1 0 48077.365 3.20
341  *
342  * EC2 - Echo Sounding (dual frequency)
343  * ------------------------------------
344  * EC2 dn t rd1 rd2
345  * dn: device number
346  * t: time tag (seconds past midnight)
347  * rd1: raw depth 1
348  * rd2: raw depth 2
349  *
350  * Example:
351  * EC2 0 48077.365 3.20 3.15
352  *
353  * ECM - Echo Soundings (multiple transducer system)
354  * -------------------------------------------------
355  * ECM dn t n rd1 rd2 ... rdn
356  * dn: device number
357  * t: time tag (seconds past midnight)
358  * n: number of transducers
359  * rd1: raw depth 1
360  * rd2: raw depth 2
361  * ...
362  * rdn: raw depth, transducer n
363  *
364  * Example:
365  * ECM 1 57274.82 9 11 10.8 10.7 11.4 11.8 13 15.1 15.5 15.6
366  *
367  * GYR - Gyro Data (Heading)
368  * -------------------------
369  * GYR dn t h
370  * dn: device number
371  * t: time tag (seconds past midnight)
372  * h: ship heading angle
373  *
374  * Example:
375  * GYR 0 57274.04 193
376  *
377  * POS - Position
378  * --------------
379  * POS dn t x y
380  * dn: device number
381  * t: time tag (seconds past midnight)
382  * x: easting
383  * y: northing
384  *
385  * Example:
386  * POS 0 57274.04 5569070.02 3774080.46
387  *
388  * ROX - Roxann data
389  * -----------------
390  * ROX dn t n e1 e2
391  * dn: device number
392  * t: time tag (seconds past midnight)
393  * n: number of values to follow (always 2)
394  * e1: roxann e1 measurement
395  * e2: roxann e2 measurement
396  *
397  * Example:
398  * ROX 2 48077.474 2 0.03 0.13
399  *
400  * SB2 - Multibeam data
401  * --------------------
402  * SB2 dn t n sv r1 r2 r3 ... rn q1 q2 ... qn
403  * dn: device number
404  * t: time tag (seconds past midnight)
405  * n: number of values to follow. Depends on device type.
406  * sv: sound velocity from device.
407  * r1-n: ranges in device units.
408  * q1-n: quality codes (0 to 3 range, 0=bad). Packed 4 per number.
409  *
410  * Example (Echoscan II):
411  * SB2 1 48077.474 39 1500.00 19.50 19.31 ...
412  *
413  * Example (Seabat 9001):
414  * SB2 1 48077.474 76 1500.00 19.50 19.31 ...
415  *
416  * Example (Seabat 9003):
417  * SB2 1 48077.474 51 1500.00 19.50 19.31 ...
418  *
419  * Example (Seabat 8101 using 101 beams):
420  * SB2 1 48077.474 51 1500.00 19.50 19.31 ...
421  *
422  */
