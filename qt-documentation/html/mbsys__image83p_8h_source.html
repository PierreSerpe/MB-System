26  * to store data from Imagenex DeltaT multibeam sonar systems.
27  * The data formats which are commonly used to store Imagenex DeltaT
28  * data in files include
29  * MBF_IMAGE83P : MBIO ID 191
30  * MBF_IMAGEMBA : MBIO ID 192
31  *
32  * Author: Vivek Reddy, Santa Clara University (initial version)
33  * D.W. Caress (many revisions since)
34  * Date: May 5, 2008

38 /*
39  * Notes on the MBSYS_IMAGE83P data structure:
40  * 1. Imagenex DeltaT multibeam systems output raw data in a format
41  * combining ascii and binary values.
42  * 2. These systems output up to 480 beams of bathymetry
43  * 3. The data structure defined below includes all of the values
44  * which are passed in the 83P Imagenex data format records plus many
45  * values calculated from the raw data.
46  * 4. The initial 83P format version was labeled 1.xx but is coded as 1.00. The
47  * second format version is 1.10. As of November 2022, versions through 1.10
48  * are supported as format MBF_IMAGE83 (191).
49  * 5. Support for comment records is specific to MB-System.
50  * 6. The MBF_IMAGE83P format does not support beam flags. Support for beam
51  * flags is specific to the extended MB-System format MBF_IMAGEMBA (id=192).
52  * Format MBF_IMAGEMBA records also include the bathymetry soundings
53  * calculated as arrays of bathymetry values and the acrosstrack and
54  * alongtrack positions of the soundings.
55  * 7. Both formats have two spaces for recording heading, roll, and pitch. If the
56  * multibeam has its own attitude sensor then these values are recorded with
57  * 0.1 degree precision. There are other spaces in the header for heading,
58  * roll and pitch stored as floats so that there are several digits of
59  * precision available. In some installations the logged files include
60  * attitude data in those secondary fields from an external sensor (and in
61  * that case can also include heave). MB-System uses the float attitude values
62  * in processing. When reading a file, if the internal integer values are
63  * nonzero and the external float values are flagged as undefined, then the
64  * former values (converted to degrees) are copied to the latter.
65  * Subsequently the external float fields are used as the source for heading
66  * and attitude data.
67  * 8. The vendor MBF_IMAGE83P format does not include a field for sonar depth, but does
68  * include a field for heave. The extended MBF_IMAGEMBA format includes separate
69  * float fields for both heave and sonar depth - the sonar depth is typically used
70  * either as a static draft on a surface vessel or a pressure depth on a
71  * submerged AUV or ROV platform. Heave is positive up and sonar depth is
72  * positive down. In some cases on submerged platforms the pressure depth is
73  * recorded into the heave field. In that case the --kluge-sensordepth-from-heave
74  * argument to mbpreprocess will cause the heave value to be moved to the
75  * sonar_depth field in the output MBF_IMAGEMBA format files.
76  * 9. Comment records are supported for both formats - this is specific to MB-System.
77  */

79 /* The following .83P format specification information is taken from a
80  * 16 March 2010 document from Imagenex Technology Corporation.
81  *
82  * DeltaT - 83P PROFILE POINT OUTPUT
83  * ( 83P UDP/IP Ethernet Datagram, .83P File Format )
84  *
85  * For each ping, the following bytes are output during the 83P UDP datagram. If
86  * recording to a .83P file, the following bytes are appended and saved to the file
87  * for each ping. The total number of bytes ‘N’ for each ping will vary depending
88  * on the number of beams selected.
89  *
90  * Byte # Byte Description
91  * ------ ----------------
92  * 0-255 File Header (256 bytes)
93  * 256-nnn Profile Ranges for current ping (2 range bytes / beam)
94  * nnn = 256 + (2*number_of_beams) – 1
95  * If Intensity Bytes are included (Byte 117 = 1),
96  * nnn = 256 + (4*number_of_beams) – 1
97  *
98  * FILE HEADER (note -this is really a record header present for each survey record)
99  * Bytes 0 through 255 contain the following File Header information:
100  *
101  * 0 ASCII '8'
102  * 1 ASCII '3'
103  * 2 ASCII 'P'
104  *
105  * 3 .83P File Version
106  * 0 = v1.00
107  * 10 = v1.10
108  *
109  * 4-5 Total Bytes ‘N’ - number of bytes that are written to the disk for this ping
110  * Byte 117 = 0 (No Intensity) ==> N = 256 + (2*number_of_beams)
111  * Byte 117 = 1 (Intensity) ==> N = 256 + (4*number_of_beams)
112  *
113  * 6 Reserved - always 0
114  * 7 Reserved - always 0
115  *
116  * 8-19 Sonar Ping Interrogation Timestamp
117  * Date – system date, null terminated string (12 bytes) "DD-MMM-YYYY"
118  *
119  * 20-28 Sonar Ping Interrogation Timestamp
120  * Time – system time, null terminated string (9 bytes) "HH:MM:SS"
121  *
122  * 29-32 Sonar Ping Interrogation Timestamp
123  * Hundredths of Seconds – system time, null terminated string (4 bytes) ".hh"
124  * Note: see Bytes 112-116 for Milliseconds.
125  *
126  * 33-46 GNSS Ships Position Latitude – text string (14 bytes) “_dd.mm.xxxxx_N”
127  * dd = Degrees
128  * mm = Minutes
129  * xxxxx = Decimal Minutes
130  * _ = Space
131  * N = North or S = South
132  *
133  * 47-60 GNSS Ships Position Longitude – text string (14 bytes) “ddd.mm.xxxxx_E”
134  * ddd = Degrees
135  * mm = Minutes
136  * xxxxx = Decimal Minutes
137  * _ = Space
138  * E = East or W = West
139  *
140  * 61 GNSS Ships Speed
141  * Speed = (Byte 61)/10 in knots
142  *
143  * 62-63 GNSS Ships Course
144  * | byte 62 | byte 63 |
145  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
146  * | Course * 10 (in degrees) |
147  *
148  * 64-65 Pitch Angle (from Internal Sensor)
149  * | byte 64 | byte 65 |
150  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
151  * | P | (Pitch Angle*10) + 900 |
152  * If 'P' = 0, Pitch Angle = 0 degrees
153  * If 'P' = 1, Pitch Angle = [[((Byte 64 & 0x7F)<<8) | (Byte 65)]-900]/10
154  *
155  * 66-67 Roll Angle (from Internal Sensor)
156  * | byte 66 | byte 67 |
157  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
158  * | R | (Roll Angle*10) + 900 |
159  * If 'R' = 0, Roll Angle = 0 degrees
160  * If 'R' = 1, Roll Angle = [[((Byte 66 & 0x7F)<<8) | (Byte 67)]-900]/10
161  *
162  * 68-69 Heading Angle (from Internal Sensor)
163  * | byte 68 | byte 69 |
164  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
165  * | H | Heading Angle*10 |
166  * If 'H' = 0, Heading Angle = 0 degrees
167  * If 'H' = 1, Heading Angle = [((Byte 68 & 0x7F)<<8) | (Byte 69)]/10
168  *
169  * 70-71 Beams
170  * | byte 70 | byte 71 |
171  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
172  * | Number of Beams |
173  *
174  * 72-73 Samples Per Beam
175  * | byte 72 | byte 73 |
176  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
177  * | Number of Samples Per Beam |
178  *
179  * 74-75 Sector Size
180  * | byte 74 | byte 75 |
181  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
182  * | Sector Size (in degrees) |
183  *
184  * 76-77 Start Angle (Beam 0 angle)
185  * | byte 76 | byte 77 |
186  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
187  * | [Start Angle (in degrees) + 180] * 100 |
188  *
189  * 78 Angle Increment
190  * Angle spacing per beam = (Byte 78)/100 in degrees
191  *
192  * 79-80 Acoustic Range
193  * | byte 79 | byte 80 |
194  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
195  * | Acoustic Range (in meters) |
196  *
197  * 81-82 Acoustic Frequency
198  * | byte 81 | byte 82 |
199  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
200  * | Acoustic Frequency (in kHz) |
201  *
202  * 83-84 Sound Velocity
203  * | byte 83 | byte 84 |
204  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
205  * | V | Sound Velocity (in meters/second) * 10 |
206  * If 'V' = 0, Sound Velocity = 1500.0 m/s
207  * If 'V' = 1, Sound Velocity = [((Byte 83 & 0x7F)<<8) | (Byte 84)]/10.0
208  *
209  * 85-86 Range Resolution
210  * | byte 85 | byte 86 |
211  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
212  * | Range Resolution (in millimeters) |
213  *
214  * 87-88 Reserved
215  * Reserved – always 0
216  *
217  * 89-90 Profile Tilt Angle (mounting offset)
218  * | byte 89 | byte 90 |
219  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
220  * | Profile Tilt Angle (in degrees) + 180 |
221  *
222  * 91-92 Repetition Rate – Time between pings
223  * | byte 91 | byte 92 |
224  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
225  * | Repetition Rate (in milliseconds) |
226  *
227  * 93-96 Ping Number – increment for every ping
228  * | byte 93 | byte 94 | byte 93 | byte 94 |
229  * | Ping Number |
230  *
231  * Version 1.00 - end of nonzero parameters in the header
232  * 97-255 Reserved – always 0
233  *
234  * Version 1.10 - bytes 100-154 now used for additional parameters
235  *
236  * 97-98 Reserved – always 0
237  *
238  * 100-103 Sonar X-Offset – 4-byte single precision floating point number
239  * | byte 100 | byte 101 | byte 102 | byte 103 |
240  * | Sonar X-Offset (in meters) |
241  *
242  * 104-107 Sonar Y-Offset – 4-byte single precision floating point number
243  * | byte 100 | byte 101 | byte 102 | byte 103 |
244  * | Sonar Y-Offset (in meters) |
245  *
246  * 108-111 Sonar Z-Offset – 4-byte single precision floating point number
247  * | byte 100 | byte 101 | byte 102 | byte 103 |
248  * | Sonar Z-Offset (in meters) |
249  *
250  * 112-116 Sonar Ping Interrogation Timestamp
251  * Milliseconds – system time, null terminated string (5 bytes) ".mmm"
252  *
253  * 117 Intensity Bytes Included
254  * 0 = No
255  * 1 = Yes
256  *
257  * 118-119 Ping Latency – Time from sonar ping interrogation to actual ping
258  * | byte 91 | byte 92 |
259  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
260  * | Ping Latency (in units of 100 microseconds) |
261  *
262  * 120-121 Data Latency – Time from sonar ping interrogation to 83P UDP datagram
263  * | byte 91 | byte 92 |
264  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
265  * | Data Latency (in units of 100 microseconds) |
266  * Time Since Ping = Data Latency – Ping Latency
267  * Note: Data Latency is not available during file playback.
268  *
269  * 122 Sample Rate
270  * 0 = Standard Resolution (1 in 500)
271  * 1 = High Resolution (1 in 5000)
272  *
273  * 123 Option Flags
274  * Bit 0 – 1 = data is corrected for roll
275  * Bit 1 – 1 = data is corrected for ray bending
276  * Bit 2 – 1 = sonar is operating in overlapped mode Bit 3 – 0
277  * Bit 4 – 0
278  * Bit 5 – 0
279  * Bit 6 – 0
280  * Bit 7 – 0
281  *
282  * 124 Reserved - always 0
283  *
284  * 125 Number of Pings Averaged
285  * 0 to 25
286  *
287  * 126-127 Center Ping Time Offset
288  * The Sonar Ping Interrogation Timestamp (Bytes 8-19, 20-28 and 112-116)
289  * is the timestamp for the current ping. But due to ping averaging,
290  * the ping time of the center ping (of a group of averaged pings)
291  * may be required (i.e. for roll stabilization). The Center Ping Time
292  * Offset is the time difference between the center ping interrogation
293  * and the current ping interrogation.
294  * | byte 126 | byte 127 |
295  * | 7 6 5 4 3 2 1 0 | 7 6 5 4 3 2 1 0 |
296  * | Center Ping Time Offset (in units of 100 microseconds) |
297  * Center Ping Time = Sonar Ping Interrogation Timestamp
298  * – Center Ping Time Offset + Ping Latency
299  * Note: Profile data from the current ping should be used when
300  * subtracting the Center Ping Time Offset.
301  *
302  * 128-131 Heave (from External Sensor)
303  * 4-byte single precision floating point number
304  * | byte 128 | byte 129 | byte 130 | byte 131 |
305  * | Heave (in meters) |
306  *
307  * 132 User Defined Byte – this is a copy of the 837 User Defined Byte
308  (Byte 45 from the .837 File Header)
309  *
310  * 133-136 Altitude
311  * 4-byte single precision floating point number
312  * | byte 133 | byte 134 | byte 135 | byte 136 |
313  * | Altitude (in meters) |
314  *
315  * 137 External Sensor Flags
316  * Bit 0 – 1 = external heading angle available
317  * Bit 1 – 1 = external roll angle available
318  * Bit 2 – 1 = external pitch angle available
319  * Bit 3 – 1 = external heave available
320  * Bit 4 – 0
321  * Bit 5 – 0
322  * Bit 6 – 0
323  * Bit 7 – 0
324  *
325  * 138-141 Pitch Angle (from External Sensor)
326  * 4-byte single precision floating point number
327  * | byte 138 | byte 139 | byte 140 | byte 141 |
328  * | Pitch (in degrees) |
329  *
330  * 142-145 Roll Angle (from External Sensor)
331  * 4-byte single precision floating point number
332  * | byte 142 | byte 143 | byte 144 | byte 145 |
333  * | Roll (in degrees) |
334  *
335  * 146-149 Heading Angle (from External Sensor)
336  * 4-byte single precision floating point number
337  * | byte 146 | byte 147 | byte 148 | byte 149 |
338  * | Heading (in degrees) |
339  *
340  * 150 Transmit Scan Flag
341  * 0 = manual scan
342  * 1 = auto-scan
343  *
344  * 151-154 Transmit Scan Angle
345  * 4-byte single precision floating point number
346  * | byte 151 | byte 152 | byte 153 | byte 154 |
347  * | Transmit Scan Angle (in degrees) |
348  *
349  * 155-255 Reserved
350  * always 0
351  *
352  * BEAM DATA
353  * Version 1.00: Only the beam ranges are provided, as an array of 2-byte
354  * unsigned integers in units of range resolution (bytes 85-86) assuming
355  * a 1500 m/s water sound speed. The number of beams is set in the 2-byte
356  * integer in bytes 70-71.
357  *
358  * Version 1.10: If byte 117 is true, then following the array of beam ranges

361  * of beams is set in the 2-byte integer in bytes 70-71.
